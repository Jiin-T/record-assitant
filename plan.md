# 생활기록부 생성 도우미 기획서

## 1. 프로젝트 개요

### 프로젝트명

생활기록부 생성 도우미

### 추진 배경

학교 현장에서 자율활동 및 진로활동 특기사항은 다양한 활동 내용을 학생별로 정리하여 생활기록부 입력 형식에 맞게 재구성해야 한다.
이 과정에서 활동명 정리, 입력 순서 조정, 내용 병합, 바이트 수 확인, 부족한 분량 보완 등의 작업이 반복적으로 발생하여 업무 부담이 크다.

이에 따라 **구글스프레드시트를 데이터베이스로 활용**하고, **웹 기반 인터페이스**를 통해 학생별 자율활동 및 진로활동 내용을 자동 정리하여 생활기록부 입력을 지원하는 도구를 구축하고자 한다.

---

## 2. 프로젝트 목표

1. 자율활동, 진로활동 데이터를 학생별로 자동 정리하여 한 화면에서 확인할 수 있도록 한다.
2. 학생별 활동명의 순서를 자유롭게 조정할 수 있도록 한다.
3. 각 입력내용의 바이트 수 및 합계를 자동 계산하여 생활기록부 작성 기준에 맞게 관리할 수 있도록 한다.
4. 부족한 바이트 수를 기준으로 학교 공통활동 문구를 추천하여 선택 입력할 수 있도록 한다.
5. 교사가 최종 검토 후 복사 또는 다운로드하여 바로 활용할 수 있도록 한다.

---

## 3. 기대 효과

* 학생별 자율활동 및 진로활동 입력 내용 정리 시간 단축
* 생활기록부 입력 형식의 일관성 확보
* 바이트 수 초과 또는 부족 문제를 사전에 점검 가능
* 학교 공통활동 문구 활용을 통한 내용 보완 효율 향상
* 교사의 수작업 부담 감소 및 오류 최소화

## 4. 시스템 구성 개요

본 시스템은 다음과 같이 구성한다.

### 4-1. 데이터 저장 방식

* **구글스프레드시트**를 데이터베이스로 활용
* 웹 애플리케이션에서 스프레드시트 데이터를 읽고 가공하여 화면에 출력

### 4-2. 데이터베이스 구성

#### 데이터베이스 1: 학생별 활동 내용 통합 DB

자율활동과 진로활동의 모든 개별 활동 데이터를 저장하는 데이터베이스

주요 항목 예시

* 학년
* 반
* 번호
* 이름
* 영역 구분(자율활동 / 진로활동)
* 활동명
* 입력내용
* 바이트 수
* 작성일 또는 수집일
* 비고

#### 데이터베이스 2: 학교 공통활동 문구 DB

학교 공통활동 내용을 바이트 수에 따라 분류하여 저장하는 데이터베이스

주요 항목 예시

* 영역 구분(자율활동 / 진로활동 / 공통)
* 활동명 또는 분류명
* 추천 문구
* 바이트 수
* 키워드
* 사용 가능 조건
* 우선순위

---

## 5. 주요 기능

## 5-1. 학생별 활동 내용 자동 정리

* 학생을 선택하면 해당 학생의 자율활동, 진로활동 데이터가 자동 조회됨
* 자율활동명과 진로활동명이 각각 구분되어 표시됨
* 동일 학생에 대한 여러 활동 내용을 영역별로 모아서 보여줌
* 생활기록부 입력 형식에 맞는 구조로 정리하여 출력함

예시 출력 항목

* 학년 / 반 / 번호 / 이름
* 자율 활동명 / 입력내용 / Byte
* 진로 활동명 / 입력내용 / Byte

---

## 5-2. 학생별 활동명 순서 조정 기능

* 학생별로 자율활동과 진로활동의 활동명 순서를 개별 조정 가능
* 드래그 앤 드롭 또는 순서 번호 입력 방식으로 구현 가능
* 순서 변경 시 최종 출력 순서에도 동일하게 반영
* 학생마다 서로 다른 활동 순서를 가질 수 있도록 저장

필요한 이유

* 학생별 강점이나 강조하고 싶은 활동이 다르므로 획일적인 순서보다 개인별 조정이 필요함

---

## 5-3. 입력내용 자동 병합 기능

* 선택된 활동명을 기준으로 관련 입력내용을 모아 하나의 최종 문장 블록으로 구성
* 자율활동과 진로활동 각각 별도로 병합 가능
* 필요 시 활동별 문단 구분 또는 줄바꿈 없이 하나의 텍스트로 결합 가능
* 중복 문장이나 불필요한 공백 제거 기능 추가 가능

---

## 5-4. 바이트 수 자동 계산 기능

* 각 활동별 입력내용의 바이트 수 자동 계산
* 자율활동 전체 바이트 수, 진로활동 전체 바이트 수를 각각 합산 표시
* 생활기록부 입력 기준에 맞게 바이트 초과 또는 부족 여부를 시각적으로 안내

표시 방식 예시

* 현재 바이트 수
* 권장 바이트 수
* 남은 바이트 수
* 초과 여부 표시

예시

* 자율활동: 1240 Byte / 권장 1500 Byte / 남은 260 Byte
* 진로활동: 1680 Byte / 권장 1500 Byte / 180 Byte 초과

---

## 5-5. 부족한 바이트 수 기반 추천 문구 기능

* 데이터베이스 2에 저장된 학교 공통활동 문구 중에서
  **현재 남은 바이트 수 이하의 문구를 자동 추천**
* 추천된 문구를 교사가 선택하면 해당 입력내용에 추가 가능
* 자율활동 또는 진로활동에 맞는 문구만 필터링하여 추천
* 바이트 수에 따라 짧은 문구, 중간 문구, 긴 문구를 구분하여 제시 가능

추천 방식 예시

* 남은 바이트 120일 경우

  * 50 Byte 문구
  * 80 Byte 문구
  * 100 Byte 문구
  * 110 Byte 문구
* 남은 바이트에 가장 근접한 순으로 정렬

추가 고려 기능

* 키워드 기반 추천
* 학생 특성 또는 활동 유형별 추천
* 자주 사용하는 문구 즐겨찾기

---

## 5-6. 최종 입력본 생성 기능

* 학생별 최종 자율활동 / 진로활동 입력본을 완성된 형태로 보여줌

## 9. 운영 절차

1. 관리자 또는 담당 교사가 데이터베이스 1에 학생별 활동 데이터를 입력 또는 업로드
2. 학교 공통활동 문구를 데이터베이스 2에 등록
3. 교사가 웹에서 학생을 검색하여 조회
4. 자율활동, 진로활동 내용을 학생별로 확인
5. 필요에 따라 활동명 순서를 조정
6. 바이트 수를 확인하고 부족한 경우 추천 문구를 선택하여 추가
7. 최종 입력본을 검토 후 복사 또는 다운로드

## 12. 유의 사항

* 생활기록부 문구의 최종 책임은 교사에게 있으므로 자동 추천 문구는 보조 수단으로 제공해야 함
* 바이트 계산 기준은 시스템 전반에서 동일하게 적용되어야 함
* 학생별 순서 조정 데이터가 덮어쓰기되지 않도록 별도 저장 구조가 필요함
* 권한 관리가 가능하다면 관리자와 일반 사용자 권한을 구분하는 것이 바람직함
